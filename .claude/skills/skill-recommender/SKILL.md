# スキル化推奨メタスキル（Skill Recommender）

## 目的

ユーザーから新たな指示を受け取った際に、それが**繰り返し発生する可能性が高いタスクか**を自動判定し、スキル化（`.claude/skills/` への SKILL.md 作成）を提案するメタスキル。

**原則: ユーザーの許可なくスキルファイルを作成しない。必ず提案→確認→作成のフローを踏む。**

---

## 判定フロー

### Step 1: スキル化スコアの算出

以下の 5 軸で 0〜2 点を採点し、合計スコアで判定する。

| 軸 | 0 点 | 1 点 | 2 点 |
|----|------|------|------|
| **反復可能性** | 一度限りの可能性が高い（例: 特定の日にだけ使う設定） | 数週間〜月に1回程度発生 | 週1回以上または日常的に発生（例: Pod再起動、helm upgrade、バックアップ確認） |
| **手順の複雑さ** | 単一コマンドで完結 | 2〜4 ステップまたは条件分岐あり | 5 ステップ以上または複数システム跨ぎ |
| **エラーリスク** | 失敗しても被害なし／簡単にリカバリ可能 | 失敗すると軽微な手戻り発生 | 失敗するとデータ損失・サービス停止など重大被害の可能性 |
| **外部依存** | 依存なし（ローカル完結） | 1〜2 の外部システム（SSH先・API など） | 3 以上の外部システムまたは連鎖的な依存 |
| **知識の陳腐化リスク** | 不変（コマンド・設定が変わらない） | 数ヶ月単位で変更の可能性 | API・バージョン・設定が頻繁に変わる／毎回調査が必要 |

### Step 2: 合計スコアによる判断

| 合計 | 判定 | アクション |
|------|------|------------|
| 0〜3 | **スキル化不要** | そのまま指示を実行 |
| 4〜6 | **スキル化を提案** | 簡易的な提案メッセージを出し、ユーザー判断を仰ぐ |
| 7〜10 | **スキル化を強く推奨** | スキル概要テンプレートを提示し、強い推奨メッセージを出す |

### Step 3: スキル化が決定した場合

ユーザーが了承した場合のみ、以下のテンプレートで `.claude/skills/<skill-name>/SKILL.md` を作成する。

---

## スキルテンプレート

```markdown
# <スキル名（日本語・簡潔に）>

## 目的
<!-- このスキルが解決する課題を 1〜2 文で -->

## 対象
<!-- 対象となる Deployment、Namespace、ファイル、システムなど -->

## 事前条件
<!-- このスキルを実行する前に満たすべき条件（例: Helm chart が最新、VPN 接続済み） -->

## 手順

### 1. <ステップ名>
```bash
# コマンド
```

### 2. <ステップ名>
```bash
# コマンド
```

<!-- 必要に応じてステップを追加 -->

## 確認
<!-- 完了を確認する方法（期待される出力例など） -->

## トラブルシュート
<!-- よくあるエラーと対処法 -->

## 参照
<!-- 関連する .agents/ ドキュメントや外部リファレンス -->
```

## ユーザーへの提案メッセージ形式

### スコア 4〜6 の場合:

```
## 💡 スキル化の提案

このタスク「<指示の要約>」は以下の理由からスキル化の恩恵が大きいと考えられます:

| 観点 | スコア | 理由 |
|------|--------|------|
| 反復可能性 | X/2 | <理由> |
| 手順の複雑さ | X/2 | <理由> |
| エラーリスク | X/2 | <理由> |
| 外部依存 | X/2 | <理由> |
| 知識陳腐化 | X/2 | <理由> |
| **合計** | **X/10** | |

スキル化する場合、`.claude/skills/<skill-name>/SKILL.md` に手順を自動生成します。
実行しますか？
```

### スコア 7〜10 の場合（強く推奨）:

```
## 🚨 スキル化を強く推奨します

このタスク「<指示の要約>」は合計 **X/10** 点で、スキル化しないリスクが高いと判断しました。

（以下、4〜6 と同じテーブル形式）

**特に注意すべき点:**
- <重大リスクの強調>

スキル化する場合、`.claude/skills/<skill-name>/SKILL.md` に手順を自動生成します。
実行しますか？
```

## 既存スキルとの重複チェック

提案前に必ず既存スキル一覧と照合し、以下を避ける:
- 全く同じ目的のスキルを重複作成しない
- 類似スキルがある場合は「既存スキルの拡張」を提案する

---

## スキル作成時の注意

1. **コマンドはすべて実行可能な形で記述**（プレースホルダー `<...>` は具体的な値に置き換え済みであること）
2. **SSH 経由の kubectl/helm は実際のホスト名・namespace を明記**
3. **プロジェクト固有のルール（日本語出力・SSH経由kubectl・pod再起動手順）をスキルにも明記**
4. **`.agents/workflows/` に既に関連手順書がある場合は、それを参照する薄いスキルにする**
