# 📝 インシデント・ポストモーテム

> **概要**
> このドキュメントは発生した事象を客観的に記録し、再発防止に繋げるためのものです。
> 犯人探しではなく、システムとプロセスの改善（Blameless Postmortem）を目的とします。

---

## 📅 基本情報

| 項目 | 内容 |
|------|------|
| 発生日時 | 202X-MM-DD HH:MM |
| 解消日時 | 202X-MM-DD HH:MM (所要時間: ◯分) |
| 影響範囲 | 例: 本番環境APIの50%が503エラー、全ユーザーのログイン不可 |
| 対応者 | @対応者 |

---

## 🔍 1. 何が起きたか（状況）

事象のサマリーと、時系列での推移を記述します。

- **タイムライン**:
  - `HH:MM`: 異常検知（アラート発報 / ユーザー指摘）

## 🔬 2. 調査プロセス

どのようなコマンドを実行し、何を確認したかをエビデンスとして残します。

### 実行コマンド例

```bash
# リソース確認
top -bn1

# ログ確認
tail -n 100 /var/log/nginx/error.log

# コンテナステータス
kubectl get pods -n default
```
